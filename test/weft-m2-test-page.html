Weft M2 test — reconstruction workflow

A controlled 6-step workflow across 3 "systems," with a built-in idle wait. Record it with Weft, then compare Weft's step list to the ground truth at the bottom.

How to run this test
  1. Make sure npm run dev is running and the Weft extension is reloaded.
  2. Open the Weft side panel → Start recording.
  3. Complete this workflow below, screen by screen, at a natural pace. Let the "Verifying…" wait finish — it's deliberately long, to test Weft's idle detection.
  4. Click Stop recording.
  5. Read Weft's step list, then compare it to the Ground truth panel at the bottom of this page.
Tip: for best results serve this over http://localhost (so path/domain populate). It still works opened as a file — segmentation is driven by clicks, submits, and the idle gap regardless.

Ground truth — the "right answer" for this workflow

Weft's reconstruction passes M2 if its steps match this list in order, with only minor errors. cross-system = a system switch Weft should flag. wait = an idle gap Weft should treat as a boundary / bottleneck.

  1. Enter applicant detailsPartner Portal · /onboarding/applicant

  6. Update tracking sheet cross-systemOps Sheet · /sheet/partners · switched systems again
What "minor errors" looks like (still a pass)
Two adjacent steps merged into one, a slightly-off label, or one extra boundary — all fine. A fail is: wrong order, a real step missing, one step exploding into many noise-events, or the 3 cross-system switches / the wait not being detected at all.
